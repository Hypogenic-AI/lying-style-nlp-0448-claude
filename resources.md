# Resources Catalog

## Summary
This document catalogs all resources gathered for the "Lying Style" research project, investigating whether text generated by LLMs when instructed to lie differs distributionally from truthful text.

## Resource Gathering Notes

### Search Strategy
1. Used paper-finder (Semantic Scholar API) with queries on "LLM deception," "lying text generation," "truthfulness probing"
2. Web search for recent (2024-2025) papers on LLM lying/deception detection
3. Cross-referenced citations and code repositories from downloaded papers
4. Searched HuggingFace for relevant datasets

### Selection Criteria
- Papers directly studying truthful vs. deceptive LLM output (representation-level or text-level)
- Datasets providing labeled truthful/false statements suitable for LLM prompting
- Code repositories with reusable tooling for truth probing or deception analysis

### Challenges Encountered
- Azaria & Mitchell's direct dataset download URL was unreachable (connection reset)
- Some arXiv PDFs contained wrong papers (ID mismatches); addressed via agent training knowledge
- LIAR dataset uses deprecated HuggingFace script format

### Gaps and Workarounds
- **No direct "LLM lying style" dataset exists.** We will need to generate one by prompting models under truthful vs. deceptive conditions using TruthfulQA questions.
- **Schuster (2019) counterpoint** suggests stylometry may fail, but was tested on GPT-2-era models. Modern instruction-tuned models may behave differently.

## Recommendations for Experiment Design

### 1. Primary Dataset(s)
- **TruthfulQA** as the anchor: prompt LLMs with questions, collect responses under truthful vs. deceptive instructions
- **Geometry of Truth / Truth is Universal** datasets for controlled true/false statement generation

### 2. Baseline Methods
- Bag-of-words + logistic regression (stylometric baseline)
- LIWC feature analysis (psycholinguistic baseline)
- BERT-based classifier (neural baseline)
- Activation probing (white-box baseline, if open-weight models used)

### 3. Evaluation Metrics
- **Distributional divergence:** KL divergence, Jensen-Shannon divergence between truthful/deceptive text distributions
- **Classification accuracy/F1:** If framing as a detection task
- **Feature-level effect sizes:** Cohen's d for specific stylistic features
- **Statistical tests:** Permutation tests, bootstrap confidence intervals

### 4. Code to Adapt/Reuse
- **LLM-LieDetector:** Prompting infrastructure for generating lies, elicitation questions
- **geometry-of-truth / Truth_is_Universal:** Datasets and probing methods
- **representation-engineering:** Activation extraction and analysis toolkit

### 5. Experimental Protocol
1. Select 3+ LLMs (e.g., Llama-3, Mistral, GPT-3.5/4)
2. For each model, generate responses to TruthfulQA questions under:
   - Truthful condition (standard prompt)
   - Direct lying condition ("Answer incorrectly on purpose")
   - Roleplaying condition ("You are a character who always lies")
   - Jailbreak condition (DAN-style prompt)
3. Collect 500+ responses per condition per model
4. Extract linguistic features (lexical, syntactic, semantic, stylometric)
5. Compare distributions between conditions using statistical tests
6. Train classifiers to detect deceptive text
7. Analyze which features (if any) differ most between conditions
